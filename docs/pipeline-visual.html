Bill to Impact: The Scoring Pipeline

How PolicyEngine goes from a tax bill to a published microsimulation estimate

1
Bill Discovery & Triage Automated
OpenStates API surfaces bills matching tax/benefit keywords. Each bill scored 0-100 for PE modelability.
Parametric (80-100)
Changes existing parameter values only (rate cut, threshold adjust)
Structural (50-79)
New brackets, new credits, restructured programs
2
Bill Research Parallel Agents
Bill Researcher
Fetch bill text, extract provisions with exact values and bill section references, identify effective dates
Fiscal Finder
Find fiscal notes, search think tanks, back-of-envelope calculation, revenue-base reasoning
3
Parameter Mapping
Map bill provisions to PolicyEngine parameter paths. Generate reform_params JSON.
{"gov.states.ga...brackets[0].rate": {"2026-01-01.2100-12-31": 0.0499}}
L1
Encoding Validation Iterative
Autonomous agent checks that reform_params correctly encodes the bill. Fixes structural errors only.
  • Parameter paths exist in PE-US?
  • Period ranges match bill effective dates?
  • All filing statuses included?
  • All provisions from bill text encoded?
  • Values match bill text exactly?
Bill values are IMMUTABLE. Never change 4.99% to match a fiscal note. Fix paths/periods/structure only.

L2
Data Diagnostic Reform-Specific
Checks driven by reform type:
Rate Cut
AGI, effective rate, top decile
EITC
Earned income, children, bottom decile

Child population, income phase-out

Findings cached in data_findings table — reused by future reforms. Produces weighted attribution of gap by variable.
6
Diagnosis & Documentation
Every gap is explained, not eliminated. Written to DB and included in PR.
encoding_correct: true
gap: 35.7% (PE: -$500M vs fiscal note: -$778M)
root_cause: "PE 2026 baseline is 5.09% (HB1015), fiscal note uses 5.19%"
data_quality_score: +0.19 (PE runs 19% higher for GA)
→ Publish with documented explanation
↻
Cross-Bill Learning Outer Loop
After N bills: detect state biases, calibrate harness strategy accuracy, build state data profiles. Each bill makes the next one faster (cached findings) and more accurate (correction factors).
↻ Correction factors and past diagnoses fed back to harness + scoring agent for next bill
